# v0.0.4 - CLI & Distribution Improvements

**Released: December 7, 2025**

This release focuses on improving the developer experience with a more robust CLI, better git hooks, and standardized distribution filenames.

## üèóÔ∏è Major Improvements

### 1. Distribution Files Renamed (Breaking Change)
To better align with the package name, the build output files have been renamed:
- **Old**: `dist/style.css` / `dist/style.min.css`
- **New**: `dist/axis-twelve.css` / `dist/axis-twelve.min.css`

> [!WARNING]
> If you are linking to the CSS files directly (e.g., via CDN or relative path), you must update your `<link>` tags to use `axis-twelve.css`.

### 2. Enhanced CLI (`./axis`)
The CLI wrapper has been significantly improved:
- **`test` command**: Added standard `test` command (alias for `check`).
- **`clean` command**: Added command to remove the `dist/` directory.
- **Dependency Checks**: CLI now verifies `node_modules` exists before running tools.
- **Argument Passthrough**: You can now pass flags to underlying tools, e.g., `./axis build --no-source-map`.

### 3. Upgraded Git Hooks
- **`pre-commit`**: 
    - Runs `lint` verification.
    - **Interactive Mode**: Asks to Bump Patch/Minor/Major (CLI only).
    - **Manual Override**: Detects if `package.json` is manually staged and skips auto-bump.
- **`pre-push`**: Now runs `npm test` (build + lint) before pushing to remote.

### 4. Version Banner
Compiled CSS files now include a standard header with version and license information.

## üìù Migration Guide

### Updating HTML Links
Search and replace references to `style.css`:

```html
<!-- Before -->
<link rel="stylesheet" href="dist/style.css">

<!-- After -->
<link rel="stylesheet" href="dist/axis-twelve.css">
```

### Updating NPM Scripts
If you reference the files in custom scripts, update the paths:

```json
"scripts": {
  "my-script": "cp node_modules/axis-twelve/dist/axis-twelve.css public/"
}
```
